1

Request assembly and normalization

Convert a user task and its context into one canonical request before any external delegation decision is made.

InputsPrompt, retrieved context, logs, code snippets, tool output, metadata.
NormalizationNormalize encodings, whitespace, code fences, hostnames, headers, and metadata fields.
OutputA canonical request object used consistently by sensors, policy, routing, audit, and evaluation.
2

Evidence providers supply facts, not authority

Reusable detectors and advisory classifiers identify risk signals. They do not decide whether a cloud target is allowed to see the request.

SecretsAPI keys, bearer tokens, credentials, config secrets.
PIIEmail, phone, person names, addresses, identifiers.
Enterprise contextInternal hostnames, topology, incident details, private code.
Adversarial textPrompt injection, evasion wording, mixed-risk requests.

4

Egress guard and sanitizer

This is the only exit path. It builds the delegated payload after policy-denied spans are removed, transformed, generalized, or summarized.

Sanitized payloadRaw secrets become placeholders; sensitive code can become pseudocode; topology can become a summary.
Audit traceDecision metadata, rule IDs, evidence classes, payload references, and leakage checks are logged.
No raw secret loggingAudit artifacts are designed to avoid storing real or synthetic raw secret values.
5

Simulated external AI endpoint

The public prototype records exactly what would have been sent outside the trusted boundary. Real OpenAI or Claude calls remain future sanitized-only smoke tests.
